# Operator Console UX v1

## 경험 목표

Operator Console은 “자동화 툴”이 아니라 “상단장님 전용 운영석”처럼 느껴져야 합니다.

핵심 인상은 아래 세 가지입니다.

- 차분하다
- 신뢰가 간다
- 중요한 정보가 바로 보인다

## 화면 구조

### 상단 바

- 현재 프로젝트명
- Codex 연결 상태
- 자동 루프 상태
- 마지막 액션 시각
- `긴급 정지` 버튼

### 좌측 패널

- 프로젝트 요약
- 목표 수준
- 운영 규칙
- 단계 목록
- 현재 단계 강조

### 중앙 메인 패널

- 현재 상태 카드
- 다음 액션 후보
- 프롬프트 미리보기
- 현재 캡처 이미지 또는 최근 관찰 정보

### 우측 패널

- 액션 타임라인
- 경고/리스크 패널
- 최근 캡처 목록
- 수동 제어 버튼 묶음

### 하단 도크

- 단축키 힌트
- 최근 이벤트 요약
- 저장 상태

## 상태 체계

### 상태 배지

- `준비`
- `관찰 중`
- `전송 가능`
- `검토 필요`
- `위험`
- `중지됨`

### 색상 원칙

- 정상: 차분한 청록 또는 블루 계열
- 주의: 앰버 계열
- 위험: 과하게 자극적이지 않은 레드 계열
- 배경: 너무 새까맣지 않은 딥 네이비 또는 소프트 다크 그레이

## 상호작용 원칙

- 자동 모드 진입 전에는 반드시 현재 타겟과 프롬프트를 보여줍니다.
- 위험 상태에서는 자동 진행보다 이유 설명을 먼저 보여줍니다.
- 수동 액션 버튼은 항상 같은 위치에 둡니다.
- 상단장님이 개입한 이력은 타임라인에서 뚜렷하게 구분합니다.

## 편의 기능

- `Ctrl+Enter` 수동 전송
- `Space` 자동 루프 일시정지/재개
- `Esc` 긴급 정지
- 최근 프로젝트 빠른 재개
- 마지막 캡처 바로 열기
- 로그 필터

## 카피 톤

- 한국어 기본
- 단정한 존댓말
- 장황한 설명보다 결론 우선
- 위험 경고는 명확하고 짧게

예시:

- “Codex 창을 다시 찾았습니다.”
- “화면이 아직 변하는 중입니다. 다음 단계는 보류합니다.”
- “다음 단계 전송 조건을 만족했습니다.”
- “위험 상태입니다. 수동 확인이 필요합니다.”

## v1에서 절대 놓치면 안 되는 UX 포인트

- 상단장님이 지금 어디까지 진행됐는지 한눈에 보여야 합니다.
- “왜 멈췄는지”를 한 줄로 이해할 수 있어야 합니다.
- 다음에 어떤 행동이 일어날지 미리 보여줘야 합니다.
- 실수했을 때 되돌릴 수 있다는 심리적 안전감이 있어야 합니다.
