# A-006: 안정화 판정 고도화

## 목적

javis가 화면이 정말 멈췄는지 더 신뢰성 있게 판단하도록 안정화 감지 로직을 개선합니다.

## 배경

현재 단순 유사도 비교만으로는 로딩 애니메이션, 커서 깜빡임, 부분적 UI 변화에 취약할 수 있습니다.

## 사용자 스토리

상단장님으로서, javis가 너무 빨리 다음 단계로 넘어가거나, 반대로 한참 기다리기만 하는 상황을 줄이고 싶습니다.

## 범위

- 안정화 판정 파라미터 정리
- 연속 안정 횟수 정책
- 쿨다운 정책
- 변화 민감 영역 보정
- false positive 로그 분류
- 판정 근거 표시

## 제외

- OCR 기반 의미 판독
- 브라우저 결과 품질 판정

## 구현 메모

- 현재 시그니처 외에 변화량 히스토리를 함께 보관
- 상태 배지에 “안정화 후보”, “불충분”, “변화 중” 노출
- 파라미터는 세션별 또는 전역 설정으로 조정 가능하게 설계

## 완료 조건

- 안정화 오탐이 체감상 줄어듦
- 왜 안정화로 판단했는지 로그에 남음
- 파라미터 조정 후 결과를 다시 관찰하기 쉬움

## 산출물

- 안정화 정책 v1
- 판정 로그 확장
- 설정값 튜닝 UI
