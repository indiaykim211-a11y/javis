# A-010: Release A 스모크 스위트

## 목적

Release A를 “돌아가는 것 같다” 수준이 아니라, 반복 검증 가능한 릴리즈로 마감할 수 있게 합니다.

## 배경

자동화 제품은 작은 회귀로도 신뢰를 잃습니다. 릴리즈 직전 점검표가 아니라, 개발 중에도 자주 돌릴 수 있는 검증 세트가 필요합니다.

## 사용자 스토리

상단장님으로서, Jarvis를 실제 프로젝트에 다시 맡기기 전에 핵심 기능이 살아있는지 빠르게 확인하고 싶습니다.

## 범위

- 핵심 시나리오 스모크 목록 정의
- 수동 검증 절차 문서화
- 릴리즈 체크리스트
- 실패 기록 양식
- 회귀 발견 시 되돌아갈 기준 정의

## 제외

- 완전 자동 E2E 테스트
- OCR/비전 검증

## 구현 메모

- 최소 스모크 시나리오 예시
  - 창 탐색
  - 캘리브레이션 재사용
  - 수동 전송
  - 자동 루프 시작/정지
  - 긴급 정지
  - 세션 복원
- 실제 프로젝트 기준으로 15분 내 점검 가능한 형태 권장

## 완료 조건

- Release A 핵심 기능을 빠르게 검증 가능
- 실패 시 어떤 티켓/영역으로 되돌아가야 할지 연결 가능
- 상단장님이 릴리즈 직전 직접 확인하기 쉬움

## 산출물

- Release A 체크리스트
- 스모크 시나리오 문서
- 실패 기록 템플릿
