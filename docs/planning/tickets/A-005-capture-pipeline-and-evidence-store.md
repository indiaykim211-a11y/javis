# A-005: 캡처 파이프라인과 증거 저장소

## 목적

javis의 판단과 행동을 나중에 되짚어볼 수 있도록 관찰 근거를 구조화합니다.

## 배경

캡처 파일만 쌓이는 것은 운영 근거가 아닙니다. 어떤 시점에, 어떤 이유로, 무엇을 근거로 찍었는지가 남아야 합니다.

## 사용자 스토리

상단장님으로서, javis가 왜 다음 단계로 넘어갔는지 캡처와 로그로 확인하고 싶습니다.

## 범위

- 캡처 이벤트 모델
- 파일명 규칙과 저장 폴더 구조
- 캡처 메타데이터 저장
- 최근 캡처 목록 UI
- 액션 로그와 캡처 연결

## 제외

- OCR 결과 저장
- 브라우저 결과 의미 해석

## 구현 메모

- 캡처 이유를 enum처럼 정리
  - manual
  - loop_observation
  - pre_send
  - failure_snapshot
- 향후 OCR 결과가 같은 레코드에 붙을 수 있게 구조 설계

## 완료 조건

- 최근 캡처를 시점/이유 기준으로 조회 가능
- 액션 로그에서 관련 캡처를 바로 찾을 수 있음
- 파일만 남는 것이 아니라 메타데이터가 함께 남음

## 산출물

- 증거 저장 스키마
- 캡처 기록 UI
- 액션-캡처 연결
