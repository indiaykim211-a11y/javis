# A-001: 창 타겟 하드닝

## 목적

javis가 Codex 창을 안정적으로 식별하고, 세션 중간에 포커스를 잃어도 다시 붙잡을 수 있게 만듭니다.

## 배경

현재 방식은 프로세스명과 제목 문자열에 많이 의존합니다. 실전에서는 창 제목 변화, 복수 창, 최소화/복구 상태 때문에 오작동 가능성이 있습니다.

## 사용자 스토리

상단장님으로서, javis가 Codex 창을 안정적으로 선택해서 엉뚱한 프로그램에 입력하지 않기를 원합니다.

## 범위

- 창 탐색 결과를 점수화하는 타겟 선택 로직
- 최근 성공 창을 우선하는 타겟 락킹
- 타겟 상실 시 재탐색 정책
- 포커스 복구 처리
- 진단 로그 추가

## 제외

- 화면 본문 OCR
- 브라우저 창 제어

## 구현 메모

- `WindowInfo` 외에 `TargetScore`, `TargetSnapshot` 모델 추가
- 제목 일치, 프로세스 일치, 최근 성공 이력, 핸들 유효성 기준으로 점수 계산
- 락된 타겟이 사라지면 재탐색 후 사용자에게 로그로 알림

## 완료 조건

- Codex 창 여러 개 중에서도 일관되게 같은 타겟을 선택
- 최소화 후 복귀해도 다시 붙잡을 수 있음
- 타겟 변경 사유가 로그에 남음

## 산출물

- 창 선택 정책 v1
- 타겟 락킹 상태 저장
- 타겟 진단 로그
