# A-003: 프롬프트 조합기와 단계 큐

## 목적

javis가 “다음에 무엇을 보낼지”를 항상 명확하게 만들고, 단계 포인터가 꼬이지 않도록 합니다.

## 배경

자동화가 불안하게 느껴지는 가장 큰 이유 중 하나는 지금 어떤 프롬프트가 생성됐는지 사용자가 알기 어렵기 때문입니다.

## 사용자 스토리

상단장님으로서, javis가 다음 단계에 어떤 문구를 보낼지 미리 보고 필요하면 수정한 뒤 진행하고 싶습니다.

## 범위

- 단계 큐 모델 정리
- 현재 단계/다음 단계 상태 표시
- 프롬프트 조합기
- 프롬프트 프리뷰 패널
- 수동 편집 후 전송
- 전송 후 포인터 증가 정책

## 제외

- 자동 QA 문구 생성
- 다중 프로젝트 큐 관리

## 구현 메모

- 시스템 문맥, 프로젝트 요약, 목표 수준, 이번 단계, 운영 규칙을 합쳐 프롬프트 생성
- 프리뷰에서 수정한 문구는 액션 로그에 원문과 함께 남김
- 포인터 증가는 실제 전송 성공 후에만 일어나도록 고정

## 완료 조건

- 다음 프롬프트가 항상 UI에서 보임
- 수동 편집 후 전송 가능
- 전송 실패 시 단계 포인터가 잘못 올라가지 않음

## 산출물

- 단계 큐 상태 모델
- 프롬프트 프리뷰 UI
- 전송 정책 정의
