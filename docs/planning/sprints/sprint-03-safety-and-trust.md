# Sprint 3: Safety and Trust

## 스프린트 목표

Jarvis를 실전에서 맡길 수 있도록 `운영 신뢰`, `안전장치`, `릴리즈 검증 체계`를 완성합니다.

## 포함 티켓

- `A-007` 운영 대시보드/타임라인
- `A-008` 긴급 정지/안전 모드
- `A-009` 자동 루프 가드레일
- `A-010` 릴리즈 스모크 스위트

## 상세 목표

### 1) 한눈에 보이는 운영 상태

- 현재 단계, 최근 액션, 다음 액션 후보, 위험 상태가 한눈에 보여야 합니다.

### 2) 사고 방지

- 언제든 즉시 멈출 수 있어야 합니다.
- 이상 상태가 감지되면 자동으로 안전 모드로 내려와야 합니다.

### 3) 자동 루프 가드레일

- 재시도 횟수 제한
- 타겟 상실 시 자동 중단
- 장시간 정지 시 보류
- 반복 실패 시 사람 호출

### 4) 릴리즈 전 검증 체계

- 최소 스모크 시나리오를 정리하고, 릴리즈 전 반복 검증 가능해야 합니다.

## 완료 조건

- 상단장님이 현재 상태를 로그를 열지 않고도 이해할 수 있음
- 위험 순간에 즉시 멈출 수 있음
- 자동 루프가 통제 불가능하게 흐르지 않음
- Release A 완료 기준을 체크리스트로 검증 가능함

## 데모 시나리오

1. 자동 루프 실행
2. 중간에 긴급 정지
3. 다시 재개
4. 의도적으로 실패 조건 주입
5. 안전 모드/보류 전환 확인
6. 스모크 체크리스트 수행

## 주요 리스크

- 상태 표시가 많아지면 오히려 복잡해질 수 있음
- 가드레일이 과하면 자동화 이점이 줄어듦
- 체크리스트가 현실 시나리오를 충분히 못 담을 수 있음
