# Sprint 2: Observability and Flow

## 스프린트 목표

javis가 단순히 클릭만 하는 것이 아니라, `지금 무엇을 보내는지`와 `왜 다음 단계로 넘어가는지`에 대한 근거를 남기게 만듭니다.

## 포함 티켓

- `A-003` 프롬프트 조합기/단계 큐
- `A-005` 캡처 파이프라인/증거 저장
- `A-006` 안정화 판정 고도화

## 상세 목표

### 1) 단계 큐 명확화

- 단계 포인터가 꼬이지 않아야 합니다.
- 다음에 보낼 프롬프트가 항상 미리 보여야 합니다.

### 2) 캡처 증거 확보

- 캡처 파일만 남기는 수준이 아니라, 언제 어떤 이유로 찍혔는지가 메타데이터로 남아야 합니다.
- 나중에 문제를 재현할 수 있어야 합니다.

### 3) 안정화 판정 품질 향상

- 단순한 이미지 유사도만으로는 부족하므로 쿨다운, 연속 안정 횟수, 변화영역 민감도 같은 보정이 필요합니다.
- false positive를 줄이는 것이 핵심입니다.

## 완료 조건

- 상단장님이 전송 전 프롬프트를 보고 안심할 수 있음
- 최근 캡처와 액션 이유가 연결되어 보임
- 자동 루프가 너무 이른 시점에 다음 단계로 넘어가지 않음

## 데모 시나리오

1. 단계 5개짜리 프로젝트 로드
2. 다음 단계 프롬프트 프리뷰 확인
3. 자동 루프 실행
4. 캡처/안정화 카운트 변화 관찰
5. 증거 파일과 로그 확인

## 주요 리스크

- 안정화 감지가 너무 보수적이면 속도가 느려짐
- 반대로 너무 공격적이면 잘못 전송할 수 있음
- 캡처 저장량이 빠르게 늘어날 수 있음
