# R1-004: Control Center 창 분리

## 목적

기존 상세 제어 기능을 메인 팝업에서 분리해 별도 설정/운영 창으로 재구성합니다.

## 배경

Release 1의 핵심은 기능 삭제가 아니라 `표면 분리`입니다. 기존 기능을 유지하되, 메인 사용자 경험을 가리지 않게 해야 합니다.

## 사용자 스토리

상단장님으로서, 복잡한 설정은 필요할 때만 별도 창에서 열고 싶습니다.

## 범위

- Control Center 별도 창 도입
- 팝업에서 Control Center 열기/닫기
- 기존 고급 기능 이동
- 좌측 내비게이션 기반 탭 구조 기본형

## 제외

- 모든 탭의 상세 완성
- 음성 설정 실제 동작

## 구현 메모

- 기존 Tkinter 콘솔을 Control Center 전용 뷰로 옮기는 방식 검토
- 팝업과 Control Center는 서로 상태를 공유하되 책임은 분리
- 작은 해상도에서 우측 패널 접기 가능성 고려

## 완료 조건

- Control Center가 별도 창으로 존재
- 고급 기능이 메인 팝업이 아닌 Control Center에 배치됨
- 팝업과 창 전환 흐름이 어색하지 않음
