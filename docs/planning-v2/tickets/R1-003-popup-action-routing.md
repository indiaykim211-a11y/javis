# R1-003: 팝업 액션 라우팅

## 목적

팝업의 핵심 버튼이 실제 제품 흐름과 연결되게 만듭니다.

## 배경

버튼이 보여도 아무 일도 안 일어나거나, 임시 더미 동작만 하면 팝업은 제품이 아니라 목업처럼 느껴집니다.

## 사용자 스토리

상단장님으로서, 팝업에서 `진행`, `보류`, `요약`, `설정`을 눌렀을 때 기대한 흐름으로 이어지길 원합니다.

## 범위

- `진행` 액션 라우팅
- `보류` 액션 라우팅
- `요약` 액션 라우팅
- `설정` 액션 라우팅
- 향후 음성/재판단 액션 슬롯 자리 확보

## 제외

- Release 2 수준의 실제 판단 자동화
- 음성 기능 실제 구현

## 구현 메모

- 기존 엔진 기능을 직접 노출하기보다 팝업 액션 계층을 거쳐 연결
- `설정`은 Control Center 열기
- `요약`은 현재 상태 기반 짧은 요약 표시 또는 확장 카드

## 완료 조건

- 핵심 버튼이 유효한 동선으로 연결됨
- 팝업에서 최소한의 조작만으로 제품 흐름을 체감할 수 있음
- 버튼 처리 계층이 Release 2에서도 유지 가능함
