# R2-005: Native vs Fallback 안전 매트릭스

## 목적

Codex 네이티브 경로와 로컬 데스크톱 제어 fallback의 경계를 명확히 해서, 제품 방향이 다시 흔들리지 않게 합니다.

## 사용자 스토리

상단장님으로서, 언제 Codex Automations를 우선 쓰고 언제 로컬 제어를 써야 하는지 앱과 문서에서 일관되게 보고 싶습니다.

## 범위

- native 우선 규칙
- fallback 조건
- 보류 / 중지 조건
- 위험 사례 메모

## 완료 조건

- 제품과 문서가 같은 안전 원칙을 공유함
