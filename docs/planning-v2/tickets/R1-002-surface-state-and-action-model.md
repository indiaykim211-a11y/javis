# R1-002: Surface State/액션 모델

## 목적

팝업에 무엇을 어떤 우선순위로 보여줄지 상태 모델을 확정합니다.

## 배경

작은 팝업은 공간이 좁기 때문에, 내부 상태를 그대로 노출하면 다시 운영 콘솔처럼 보이게 됩니다.

## 사용자 스토리

상단장님으로서, 팝업에서 현재 상태와 이유와 다음 행동만 짧게 보고 싶습니다.

## 범위

- 상태 배지 모델
- 요약 문장 모델
- 최근 판단 이유 슬롯
- 다음 행동 후보 슬롯
- 위험 레벨 모델

## 제외

- 실제 OpenAI 판단 로직
- 고급 로그/메타데이터 표시

## 구현 메모

- 현재 RuntimeState를 그대로 쓰지 말고 Surface 전용 ViewModel을 둠
- 상태 텍스트는 한국어 제품 톤 기준으로 정규화
- 버튼 노출 규칙은 상태에 따라 달라질 수 있게 설계

## 완료 조건

- 팝업 표시용 상태 구조가 코드와 문서에 명확함
- “상태/이유/다음 행동”이 분리되어 표현 가능
- Release 2에서 판단 엔진 결과를 쉽게 바인딩 가능

- Popup Action Model v1
