# R5-001: capture target planner

## 목적

지금 어떤 화면을 읽는 것이 가장 가치가 높은지 먼저 고르는 planner를 만듭니다.

## 사용자 스토리

상단장님으로서, javis가 아무 화면이나 다 읽지 말고 `지금 판단에 필요한 화면만` 보길 원합니다.

## 범위

- Codex 창 우선순위
- 브라우저 결과 화면 우선순위
- 오류 영역 / 핵심 CTA / 빈 화면 같은 관찰 포인트 선정

## 완료 조건

- 현재 상태 기준으로 무엇을 캡처할지 우선순위를 정할 수 있음
