# R3-007: automation safety guard

## 목적

automation이 과하게 권장되거나, 위험한 상황에서 무리하게 진행되지 않도록 stop / ask_user 기준을 제품 안에 고정합니다.

## 사용자 스토리

상단장님으로서, automation을 쓰면 안 되는 상황도 분명히 알고 싶습니다.

## 범위

- no automation 우선 조건
- stop / ask_user 조건
- 위험 메모

## 완료 조건

- automation을 써야 할 때와 말아야 할 때가 제품 안에 드러남
