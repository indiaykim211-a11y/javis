# R1-006: 정책 편집기 스켈레톤

## 목적

Release 2 판단 엔진 연결 전에, 상단장님이 실제로 정책을 편집할 수 있는 기본 편집기 구조를 미리 확보합니다.

## 배경

OpenAI 판단 엔진은 초반부터 붙을 예정이므로, 프롬프트 정책과 안전 규칙을 다룰 자리가 Release 1에서부터 준비돼야 합니다.

## 사용자 스토리

상단장님으로서, 나중에 상황별 운영 규칙과 판단 기준을 Control Center 안에서 직접 적고 관리하고 싶습니다.

## 범위

- 정책 탭 기본 레이아웃
- 정책별 섹션 분리
- 기본 템플릿 복원 버튼
- 저장 상태 표시
- 최근 수정 시각 또는 메모 슬롯

## 제외

- 정책 충돌 자동 분석의 완성형
- 실제 모델 요청 미리보기

## 구현 메모

- `master/progress/vision/repair/report/safety` 정책 구조를 그대로 반영
- 장문 텍스트 편집 UX 고려
- 나중에 버전 이력 기능이 들어갈 자리 확보

## 완료 조건

- 상단장님이 정책 구조를 한눈에 이해할 수 있음
- 기본 템플릿과 사용자 커스텀을 구분할 수 있음
- Release 2 구현 시 실제 엔진 연결 자리가 준비됨
