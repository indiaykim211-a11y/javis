# Release 1: Assistant Alpha

## 릴리즈 미션

`javis`를 처음으로 `어시스턴트처럼 보이고 쓸 수 있는 상태`까지 끌어올립니다.

이 릴리즈의 본질은 기능을 많이 넣는 것이 아니라, 제품의 중심을 `운영 콘솔`에서 `데스크톱 어시스턴트`로 바꾸는 것입니다.

## 이 릴리즈가 해결해야 하는 문제

현재 상태의 가장 큰 문제는 아래입니다.

- 제품이 상단장님에게 `도와주는 어시스턴트`가 아니라 `복잡한 운영 패널`처럼 보임
- 첫 실행 시 너무 많은 제어 요소가 드러나 부담이 큼
- 앞으로 OpenAI 판단, 화면 인식, 음성을 붙여도 메인 경험의 그릇이 아직 없음

Release 1은 이 문제를 해결하기 위해 `메인 UX의 그릇`을 먼저 완성합니다.

## 핵심 사용자 시나리오

1. 상단장님이 `javis`를 실행합니다.
2. 화면에는 작은 팝업 어시스턴트만 보입니다.
3. 어시스턴트는 현재 프로젝트, 현재 상태, 다음 행동 후보를 짧게 보여줍니다.
4. 상단장님은 `진행`, `보류`, `요약`, `설정 열기` 정도의 핵심 액션만 봅니다.
5. 복잡한 설정은 필요할 때만 별도 설정/운영 창에서 엽니다.
6. 앱을 다시 켜도 이전 세션과 최근 프로젝트를 이어서 볼 수 있습니다.

## Release 1의 제품 목표

### 1) 첫인상 전환

- 더 이상 개발자용 운영 콘솔처럼 보이지 않아야 합니다.
- 처음 보는 사람도 “이건 데스크톱 어시스턴트다”라고 느껴야 합니다.

### 2) 상호작용 단순화

- 메인 화면에서는 핵심 상태와 핵심 행동만 보여야 합니다.
- 복잡한 조정은 메인 화면에서 보이지 않아야 합니다.

### 3) 이후 확장 기반 확보

- 이후 `Codex 진행 제어`, `화면 인식`, `음성`이 들어갈 자리가 미리 준비돼 있어야 합니다.
- 지금부터 팝업 어시스턴트와 설정창의 역할이 분리돼야 합니다.

## 포함 범위

- 작은 팝업형 메인 창
- 설정/운영 창 분리
- 세션/최근 프로젝트 복원 진입점
- 프로젝트 상태 요약 카드
- 핵심 액션 버튼 묶음
- OpenAI 판단 엔진 연결을 위한 상태 슬롯 설계
- 음성 기능을 나중에 넣을 수 있는 자리 확보

## 제외 범위

- 본격적인 Codex 자동 진행 판단
- OCR/비전 기반 화면 판독
- 음성 인식 및 음성 응답 실제 구현
- 웨이크워드
- 장기 자율 운영 로직 고도화

## 필수 기능 기준

### 메인 팝업

- 항상 위 고정 가능
- 컴팩트한 크기
- 현재 상태 한 줄 요약
- 현재 프로젝트명 또는 프로젝트 요약
- 다음 행동 후보
- `진행`, `보류`, `요약`, `설정` 버튼

### 설정/운영 창

- 기존 고급 제어를 여기에 수용
- 프롬프트 정책 입력
- Codex 타겟/캘리브레이션
- 로그/스크린샷/세션 복원

### 상태 복원

- 마지막 세션 자동 복원
- 최근 프로젝트 목록
- 현재 단계 인덱스 재사용
- 마지막 캡처/로그 참조 보기

## 성공 기준

### 사용자 경험 기준

- 메인 화면만 보고도 지금 상태를 대략 이해할 수 있음
- 처음 봤을 때 메뉴 과다 느낌이 크게 줄어듦
- 설정창은 필요할 때만 열면 됨

### 제품 기준

- 팝업 어시스턴트와 설정창 역할이 명확히 분리됨
- 이후 Release 2에서 판단 엔진을 무리 없이 끼워 넣을 수 있음
- 기존 세션 저장/복원 흐름이 유지됨

### 디자인 기준

- 작은 크기에서도 정보 밀도가 좋음
- 지나치게 개발 툴처럼 보이지 않음
- 한국어 제품 톤이 자연스러움

## 주요 산출물

- Assistant Surface v1
- Control Center v1
- Session Resume Entry v1
- Release 2 연동 슬롯 정의

## 주요 리스크

### 1) 기능을 숨기다가 오히려 답답해질 수 있음

- 해결 방향: 메인 화면은 단순하게, 설정창은 충분히 강하게

### 2) 팝업이 너무 작아 정보가 부족할 수 있음

- 해결 방향: 기본은 컴팩트, 필요 시 확장 가능한 2단 레이아웃 검토

### 3) 기존 콘솔 기능이 분산되며 유지보수가 복잡해질 수 있음

- 해결 방향: 기능 제거가 아니라 `표면 분리` 관점으로 설계

## Release 1 완료 선언 조건

1. 메인 팝업만으로 기본 상태 이해가 가능함
2. 설정/운영 창에서 기존 상세 제어 접근이 가능함
3. 세션 복원과 최근 프로젝트 진입이 자연스럽게 이어짐
4. 상단장님 기준으로 “이제 어시스턴트처럼 보인다”는 평가를 받을 수 있음
