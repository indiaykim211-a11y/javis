# Release 1 Epics

## Epic R1-A. Assistant Surface Core

### 목적

`javis`가 처음으로 데스크톱 어시스턴트처럼 보이고, 작은 팝업 안에서 상태와 핵심 액션을 제공하게 만듭니다.

### 주요 산출물

- Assistant Popup 기본 창
- Surface State 모델
- 팝업 액션 바
- 상태/이유/다음 행동 카드

### 포함 티켓

- `R1-001`
- `R1-002`
- `R1-003`

## Epic R1-B. Control Center Split

### 목적

기존 복잡한 운영 콘솔을 메인 화면에서 분리하고, 설정/운영 전용 백오피스 화면으로 재구성합니다.

### 주요 산출물

- Control Center 전용 창
- 탭 기반 정보구조
- 정책 편집 기본 틀

### 포함 티켓

- `R1-004`
- `R1-006`

## Epic R1-C. Session Resume Experience

### 목적

앱을 다시 켰을 때 “이어서 쓰는 제품”처럼 느껴지도록 프로젝트/세션 복원 흐름을 전면에 배치합니다.

### 주요 산출물

- 프로젝트 홈 탭
- 최근 프로젝트 복원 진입점
- 현재 단계/최근 판단/마지막 저장 상태 요약

### 포함 티켓

- `R1-005`

## Epic R1-D. Visual Finish & Release Readiness

### 목적

작은 팝업 제품으로서의 완성도를 올리고, Release 1을 반복 검증 가능한 상태로 마감합니다.

### 주요 산출물

- 컴팩트 동작 규칙
- 시각적 완성도 보정
- Release 1 스모크 스위트

### 포함 티켓

- `R1-007`
- `R1-008`

## 에픽 순서

권장 순서는 아래와 같습니다.

1. `R1-A`
2. `R1-B`
3. `R1-C`
4. `R1-D`

이 순서의 이유는 메인 팝업의 존재감이 먼저 살아야 Control Center 분리의 가치가 생기고, 그다음에야 세션 복원과 시각적 완성도를 제대로 평가할 수 있기 때문입니다.
