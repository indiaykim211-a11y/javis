# Release 3 Epics

## Epic R3-A. Automation Authoring

### 목적

`javis`가 현재 프로젝트에 맞는 automation 방식을 고르고, Codex 앱에 바로 옮길 수 있는 launch-ready 초안을 준비하게 만듭니다.

### 주요 산출물

- automation mode selector
- thread automation composer
- project automation composer

### 포함 티켓

- `R3-001`
- `R3-002`
- `R3-003`

## Epic R3-B. Automation Operations

### 목적

Codex app에서 automation을 만든 뒤, 상단장님이 `지금 무엇이 실행 중이고 어디를 다시 봐야 하는지` 이해할 수 있게 만듭니다.

### 주요 산출물

- launch / handoff 패널
- automation runboard
- triage summary bridge

### 포함 티켓

- `R3-004`
- `R3-005`
- `R3-006`

## Epic R3-C. Safety & Readiness

### 목적

automation을 과하게 쓰지 않게 하고, `Codex-first` 원칙을 유지한 채 Release 3를 반복 검증 가능한 상태로 마감합니다.

### 주요 산출물

- automation safety guard
- Release 3 smoke suite

### 포함 티켓

- `R3-007`
- `R3-008`

## 에픽 순서

1. `R3-A`
2. `R3-B`
3. `R3-C`

이 순서의 이유는 먼저 `어떤 automation을 어떻게 만들지`가 정리돼야, 그다음에야 `운영과 재진입`을 제품적으로 다룰 수 있기 때문입니다.
