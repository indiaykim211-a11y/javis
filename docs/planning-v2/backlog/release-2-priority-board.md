# Release 2 Priority Board

## 우선순위 기준

- `P0`
  - Release 2의 정체성을 결정하는 필수 항목
- `P1`
  - Release 3 연결성과 운영 품질을 크게 높이는 핵심 항목
- `P2`
  - 있으면 좋지만 일부는 다음 릴리즈로 이월 가능한 항목

## 우선순위 보드

## 구현 순서 메모

- `R2-001`은 Release 2 전체 방향 전환의 시작점이라 가장 먼저 필요합니다.
- `R2-002`가 있어야 전략 센터가 단순 설명 패널이 아니라 실제 운영 준비 도구가 됩니다.
- `R2-003`, `R2-004`는 프로젝트별 Codex 운영을 반복 가능한 체계로 만들어 줍니다.
- `R2-005`, `R2-006`이 있어야 Phase 2가 다시 클릭 매크로 중심으로 흔들리지 않습니다.
