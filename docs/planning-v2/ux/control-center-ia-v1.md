# Control Center IA v1

## 문서 목적

이 문서는 `javis`의 설정/운영 창인 `Control Center`의 정보구조를 정의합니다.

핵심 목적은 아래와 같습니다.

- 메인 어시스턴트 팝업에서 숨겨야 할 복잡한 기능을 어디에 어떻게 배치할지 정하기
- 상단장님이 실제 운영 중 가장 자주 만질 항목과 가끔만 만질 항목을 구분하기
- 이후 OpenAI 판단, 화면 인식, 음성 옵션이 늘어나도 창이 무너지지 않게 하기

## Control Center의 역할

Control Center는 `주인공 화면`이 아니라 `백오피스`입니다.

Assistant Popup이 아래 역할을 담당한다면:

- 현재 상태 보여주기
- 왜 멈췄는지 알려주기
- 지금 뭘 하면 되는지 제안하기

Control Center는 아래 역할을 담당합니다.

- 정책을 설정하기
- 현재 세션과 최근 프로젝트를 관리하기
- Codex 제어를 조정하기
- 로그와 증거를 자세히 보기
- 자동화와 안전 규칙을 조정하기
- 나중에는 음성과 모델 옵션을 관리하기

## 설계 원칙

### 1) 메인 화면을 대신하지 않기

- 상태 요약은 팝업이 담당
- Control Center는 깊이 들어가는 화면이어야 함

### 2) 사용 빈도 순서로 정렬하기

- 자주 쓰는 탭이 앞에 와야 함
- 드물게 쓰는 고급 옵션은 뒤로 보내야 함

### 3) 판단 관련 설정을 한곳에 모으기

- 프롬프트 정책
- 판단 규칙
- 안전 규칙

이 셋은 서로 멀리 떨어지면 안 됩니다.

### 4) 읽기와 조정을 구분하기

- “현재 무슨 일이 벌어지고 있는지” 보는 구역
- “어떻게 행동할지 정책을 바꾸는” 구역

둘을 명확히 나눠야 합니다.

## 창 전체 구조

Control Center는 아래 4영역 구조를 권장합니다.

### 1) 상단 헤더

- 현재 프로젝트명
- 마지막 저장 시각
- 현재 세션 상태
- Codex 연결 상태
- OpenAI 판단 엔진 상태
- `저장`, `닫기`, `팝업으로 돌아가기`

### 2) 좌측 내비게이션 레일

탭을 세로형으로 배치합니다.

권장 순서:

1. `프로젝트`
2. `정책`
3. `Codex 제어`
4. `증거/로그`
5. `안전/자동화`
6. `모델/음성`
7. `고급`

### 3) 중앙 메인 작업 영역

- 현재 탭의 주된 설정과 정보를 보여줌
- 대부분의 편집이 여기서 일어남

### 4) 우측 보조 패널

- 현재 선택 항목 설명
- 최근 변경 이력
- 관련 경고
- 추천값 또는 기본 템플릿

작은 해상도에서는 우측 패널을 접을 수 있어야 합니다.

## 탭 구조

### 탭 1. 프로젝트

가장 자주 열게 될 시작 탭입니다.

#### 목적

- 현재 프로젝트와 세션을 한눈에 보기
- 최근 프로젝트를 다시 열기
- 현재 단계와 진행 상태를 빠르게 확인하기

#### 포함 요소

- 현재 프로젝트 요약
- 목표 수준
- 현재 단계 인덱스
- 전체 단계 수
- 최근 프로젝트 목록
- 마지막 캡처 참조
- 마지막 판단 결과 요약

#### 액션

- 현재 프로젝트 저장
- 최근 프로젝트 불러오기
- 새 프로젝트 시작
- 현재 단계로 점프

### 탭 2. 정책

상단장님이 가장 중요한 기준을 조정하는 탭입니다.

#### 목적

- OpenAI 판단 엔진에 들어가는 정책을 관리
- 운영 성향을 조정

#### 하위 섹션

- 운영 마스터 프롬프트
- 단계 진행 판단 정책
- 화면 판독 정책
- 재지시 생성 정책
- 사용자 보고 톤
- 안전 규칙

#### 액션

- 기본 템플릿 복원
- 현재 정책 저장
- 정책 버전 메모 남기기
- 정책 충돌 검사

### 탭 3. Codex 제어

현재까지 만든 로컬 제어 기능을 모으는 탭입니다.

#### 목적

- Codex 타겟 창과 입력 관련 설정 조정
- 프롬프트 프리뷰와 전송 흐름 확인

#### 포함 요소

- 창 타겟 설정
- 타겟 상태
- 입력창 캘리브레이션
- 현재 프롬프트 프리뷰
- 단계 큐
- 수동 전송/테스트 전송

#### 액션

- 창 새로고침
- Codex 포커스
- 캘리브레이션 시작
- 테스트 전송
- 프롬프트 원문 복원

### 탭 4. 증거/로그

문제가 생겼을 때 가장 많이 보게 될 탭입니다.

#### 목적

- 왜 그런 판단이 나왔는지 확인
- 현재 상태를 재현 가능한 형태로 보기

#### 포함 요소

- 최근 로그 라인
- 최근 캡처 목록
- 판단 입력 요약
- 판단 출력 요약
- anomalies 목록
- 최근 Codex 상태 요약

#### 액션

- 로그 필터
- 캡처 열기
- 최근 판단 기록 복사
- 문제 보고용 내보내기

### 탭 5. 안전/자동화

자동화 성향과 사고 방지 장치를 다루는 탭입니다.

#### 목적

- 자동화가 너무 공격적이거나 너무 소심하지 않게 조정
- 이상 상태 시 행동을 정의

#### 포함 요소

- 자동 루프 on/off 기본값
- 재시도 제한
- 확신 임계값
- high risk 시 행동
- ask_user 전환 조건
- 긴급 정지 정책

#### 액션

- 보수적/균형/공격적 프리셋 적용
- 안전 정책 저장
- 스모크 규칙 테스트

### 탭 6. 모델/음성

처음에는 얕게 시작하지만 나중에 중요해지는 탭입니다.

#### 목적

- 모델 선택과 음성 기능 확장 자리 확보

#### 포함 요소

- OpenAI 모델 선택
- 판단 엔진 응답 길이/스타일 옵션
- 음성 기능 준비 상태
- push-to-talk 옵션 자리
- TTS 출력 장치 자리

#### v1에서는

- 실제 음성 기능 구현보다 `자리 확보`가 목적
- 아직 비활성 상태여도 구조는 미리 준비

### 탭 7. 고급

자주 쓰면 안 되는 탭입니다.

#### 목적

- 디버그와 실험성 옵션 수용

#### 포함 요소

- 개발용 토글
- 진단 모드
- 내부 상태 덤프
- 실험 기능 플래그

## 추천 기본 진입 순서

처음 열었을 때 기본 탭은 `프로젝트`가 좋습니다.

이유:

- 지금 무엇을 진행 중인지 바로 보여줄 수 있음
- 최근 프로젝트 복원 동선이 자연스러움
- 초반부터 정책 편집 화면이 뜨면 부담이 큼

## 주요 사용자 흐름

### 1) 첫 설정 흐름

1. 프로젝트 탭에서 현재 프로젝트 확인
2. 정책 탭에서 기본 운영 정책 점검
3. Codex 제어 탭에서 창 타겟과 캘리브레이션 확인
4. 저장 후 팝업으로 복귀

### 2) 판단 품질 조정 흐름

1. 정책 탭 열기
2. 단계 판단 또는 안전 규칙 수정
3. 저장
4. 증거/로그 탭에서 최근 판단과 비교

### 3) 이상 상황 조사 흐름

1. 팝업에서 `설정` 진입
2. 증거/로그 탭 확인
3. anomalies와 최근 판단 확인
4. 필요 시 Codex 제어 또는 정책 탭으로 이동

### 4) 장기 세션 재개 흐름

1. 프로젝트 탭 열기
2. 최근 프로젝트 선택
3. 현재 단계와 마지막 저장 시각 확인
4. 팝업으로 돌아가 진행 재개

## 무엇을 팝업에 두지 말아야 하나

아래는 Popup이 아니라 Control Center에 있어야 합니다.

- 긴 프롬프트 편집기
- 캘리브레이션 세부 절차
- 장문의 로그
- 캡처 목록 전체
- 모델 옵션 세부값
- 실험성 토글

## 상태/저장 UX 원칙

### 1) 저장 여부를 명확히 보여주기

- 저장됨
- 저장 안 됨
- 임시 수정 중

이 구분이 필요합니다.

### 2) 탭 이동 시 손실 방지

- 정책 텍스트를 수정 중이면 경고
- 세션 전환 시 미저장 변경 경고

### 3) 최근 변경 흔적 남기기

- 마지막 수정 시각
- 최근 변경 탭
- 가능하면 간단한 변경 메모

## Release별 확장성

- 프로젝트/정책/Codex 제어/로그 중심

- 비전/OCR 관련 설정과 증거 패널 확장

- 음성 입출력 설정 실사용 수준으로 확장

## 한 줄 결론

`Control Center`는 모든 기능을 다 보여주는 창이 아니라, 상단장님이 필요할 때만 들어가서 정책을 정하고 원인을 파악하고 제어를 조정하는 `백오피스형 운영 창`이어야 합니다.
