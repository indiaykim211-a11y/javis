# Assistant Popup UX v1

## UX 목표

상단장님이 `javis`를 볼 때 느껴야 하는 첫인상은 아래와 같습니다.

- 가볍다
- 똑똑하다
- 지금 상황을 바로 알려준다
- 부담 없이 켜둘 수 있다

## 이 화면이 해야 하는 일

Assistant Popup은 모든 걸 다 보여주는 화면이 아닙니다.

반드시 해야 하는 일은 아래 4개뿐입니다.

- 현재 상태를 짧게 보여주기
- 왜 그런 상태인지 말해주기
- 다음 행동을 제안하기
- 필요할 때 설정창으로 보내기

## 레이아웃 원칙

### 1) 기본 크기

- 작고 세로형에 가까운 비율
- 화면 한쪽 구석에 놓아도 부담이 없음
- 필요 시 약간 확장 가능

### 2) 정보 우선순위

위에서 아래로 아래 순서를 유지합니다.

1. 상태
2. 한 줄 요약
3. 이유
4. 다음 행동
5. 버튼

### 3) 고급 정보 노출 방식

- 기본 상태에서는 숨김
- 눌렀을 때만 펼침
- 로그/세부설정은 팝업 안에 다 넣지 않고 Control Center로 이동

## 화면 구성안

### 헤더

- `javis` 이름 또는 아이콘
- 현재 상태 배지
- 프로젝트명 또는 짧은 프로젝트 라벨

상태 배지 예시:

- 준비
- 관찰 중
- 진행 가능
- 보류
- 위험

### 본문 카드 1: 현재 상태

예시 카피:

- “Codex 응답을 기다리는 중입니다.”
- “다음 단계 진행 조건을 확인 중입니다.”
- “지금은 수동 확인이 필요합니다.”

### 본문 카드 2: 이유

예시 카피:

- “최근 응답 변화가 아직 커서 조금 더 지켜보겠습니다.”
- “현재 단계 프롬프트는 준비됐고, 상단장님 승인만 남았습니다.”
- “반복 실패가 감지되어 자동 진행을 멈췄습니다.”

### 본문 카드 3: 다음 행동

예시 카피:

- “지금 바로 다음 단계를 진행할 수 있습니다.”
- “화면이 안정될 때까지 대기하는 것이 좋습니다.”
- “설정창에서 판단 정책을 조정해 주세요.”

### 하단 액션 바

기본 버튼은 아래 4개를 추천합니다.

- `진행`
- `보류`
- `요약`
- `설정`

후속 버전에서 추가 가능한 버튼:

- `말하기`
- `다시 판단`
- `중지`

## Control Center 진입 규칙

아래 상황에서는 팝업 내부가 아니라 Control Center로 보내는 것이 맞습니다.

- 프롬프트 정책 수정
- 로그/증거 자세히 보기
- Codex 타겟 재설정
- 캘리브레이션
- 모델/음성 옵션 변경

## 카피 원칙

- 짧고 단정한 존댓말
- 먼저 결론, 다음에 이유
- 불필요하게 기술적이지 않게
- 확신이 낮으면 낮다고 솔직하게 표현

좋은 예:

- “지금은 조금 더 기다리는 편이 안전합니다.”
- “다음 단계 진행 준비가 됐습니다.”
- “판단 확신이 낮아 수동 확인을 권장합니다.”

피해야 할 예:

- “시그니처 임계값이 기준을 충족하지 못해 보류했습니다.”
- “타겟 윈도우와 런타임 상태가 비동기적으로 어긋났습니다.”

## 시각 톤

- 툴 느낌보다 어시스턴트 느낌
- 과한 미래지향 네온 느낌은 피함
- 차분하지만 프리미엄한 인상
- 한국어 텍스트 가독성 최우선

## v1에서 반드시 살아야 하는 포인트

1. 메뉴가 많아 보이면 안 됨
2. 상태와 다음 행동이 한눈에 보여야 함
3. 상단장님이 “내가 뭘 해야 하는지” 바로 이해해야 함
4. 설정창은 있어도 메인 화면을 잡아먹으면 안 됨
