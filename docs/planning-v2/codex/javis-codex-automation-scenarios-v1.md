# javis x Codex Automation Scenarios v1

## 문서 목적

이 문서는 상단장님 실제 운영 방식에 맞춰 `Codex Automations를 어디에 어떻게 쓸지` 시나리오 단위로 정리한 문서입니다.

핵심 원칙은 아래 두 가지입니다.

- 반복 실행과 follow-up은 Codex Automations를 먼저 쓴다
- `javis`는 그 위에서 감독, 요약, 정책 적용을 맡는다

## 시나리오를 보는 방법

각 시나리오는 아래 5개 기준으로 봅니다.

- 어떤 자동화 종류가 맞는가
- Codex가 직접 맡는 일은 무엇인가
- `javis`가 위에서 덮는 일은 무엇인가
- 지금 당장 가능한가
- 언제 멈추거나 사람을 불러야 하는가

## 시나리오 1. 마스터플랜 Follow-up Heartbeat

### 추천 타입

- `thread automation`

### 상황

- 상단장님이 이미 마스터 개발 플랜을 만들어 둔 상태
- 같은 개발 스레드에서 Codex가 계속 일을 이어가길 원함
- 중간중간 다시 깨워서 상태를 보게 하고 싶음

### Codex가 맡는 일

- 같은 스레드를 다시 열고 최근 맥락 확인
- 지금 단계가 끝났는지 확인
- 막혔는지, 더 기다려야 하는지, 다음 작업이 가능한지 정리

### javis가 맡는 일

- 팝업으로 한 줄 요약 보여주기
- 정책에 따라 `진행 / 보류 / 사람 확인`으로 해석
- 나중에 화면 증거가 붙으면 판단 보강

### 장점

- 같은 문맥이 유지됩니다.
- 상단장님 방식과 가장 잘 맞습니다.

### 주의점

- 프롬프트에 `언제 멈추고 언제 보고할지`를 명확히 적어야 합니다.

## 시나리오 2. Release Smoke 자동 점검

### 추천 타입

- `project automation`

### 상황

- 일정 주기로 smoke test를 돌리고 싶음
- 우리 프로젝트처럼 `release_1_smoke.py` 같은 러너가 있음

### Codex가 맡는 일

- 정해진 스크립트 실행
- 실패/성공 요약
- 실패 시 핵심 로그와 재현 단서 정리

### javis가 맡는 일

- 상단장님에게 “지금 릴리즈 상태 괜찮음 / 보류 필요”를 짧게 보고
- 실패 시 다음 조치 추천

### 장점

- 반복 점검을 사람 손으로 안 해도 됩니다.
- 릴리즈 기준선이 생깁니다.

### 주의점

- 코드 변경이 생길 가능성이 있으면 worktree를 기본으로 봅니다.

## 시나리오 3. PR Babysitting

### 추천 타입

- `thread automation`
- 또는 `project automation + skill`

### 상황

- PR에 리뷰가 달리는지 계속 보고 싶음
- 새 피드백이 생기면 이어서 수정 루프를 돌리고 싶음

### Codex가 맡는 일

- PR 상태 확인
- 새 리뷰 코멘트 확인
- 수정이 필요한 경우 후속 작업 제안 또는 직접 작업

### javis가 맡는 일

- “리뷰 없음 / 수정 필요 / 병합 가능” 상태 배지 제공
- 중요한 변화만 팝업으로 요약

### 장점

- 웹에서 계속 새로고침하는 시간이 줄어듭니다.

### 주의점

- 어떤 피드백은 사람 판단이 꼭 필요하므로 자동 병합 같은 공격적 행동은 초반에 피합니다.

## 시나리오 4. Nightly Project Brief

### 추천 타입

- `standalone / project automation`

### 상황

- 매일 아침 프로젝트 진행 상황을 빠르게 파악하고 싶음

### Codex가 맡는 일

- 최근 커밋 / PR / 변경 흐름 요약
- workstream 단위로 브리핑 생성

### javis가 맡는 일

- 상단장님용 한국어 짧은 버전 요약
- 중요한 변화만 팝업에서 먼저 보여주기

### 장점

- 상단장님이 코드 세부 변경을 다 안 봐도 됩니다.

## 시나리오 5. Recent Code Self-Bugfix

### 추천 타입

- `project automation + skill`

### 상황

- 최근에 내가 넣은 코드에서 문제를 먼저 잡고 싶음

### Codex가 맡는 일

- 최근 변경 범위 추리기
- 관련 실패 찾기
- 직접 연결된 버그만 고치기

### javis가 맡는 일

- `자동 수정 진행 중`, `수정 후보 있음`, `사람 확인 필요` 같은 상위 상태로 보여주기

### 장점

- 나중에 터질 버그를 미리 줄일 수 있습니다.

### 주의점

- unrelated legacy issue까지 건드리지 않게 범위를 좁혀야 합니다.

## 시나리오 6. CI Failure Triage

### 추천 타입

- `project automation`

### 상황

- CI 실패를 사람이 일일이 보는 시간이 아까움

### Codex가 맡는 일

- 실패 로그 확인
- 가장 가능성 높은 원인 정리
- 관련 파일과 후속 작업 제안

### javis가 맡는 일

- 상단장님께 “빌드 실패”, “테스트 실패”, “환경 실패”처럼 한 단계 위 분류로 보여주기

### 장점

- 실패를 보는 진입 비용이 낮아집니다.

## 시나리오 7. Deploy / Release Babysit

### 추천 타입

- `thread automation`

### 상황

- 배포, 릴리즈, 외부 피드백 대기처럼 시간이 걸리는 작업을 계속 관찰해야 함

### Codex가 맡는 일

- 같은 스레드를 다시 깨워 상황 확인
- 새 로그, 새 결과, 새 피드백이 있는지 확인
- 이상 신호가 있으면 멈추고 요약

### javis가 맡는 일

- 팝업에서 `배포 감시 중`, `결과 대기`, `문제 감지`처럼 보여주기

### 장점

- 사람이 계속 붙어서 지켜보지 않아도 됩니다.

## 시나리오 8. Multi-project Morning Board

### 추천 타입

- `standalone automation`

### 상황

- 프로젝트가 여러 개일 때 아침마다 전체 상황을 한 번에 보고 싶음

### Codex가 맡는 일

- 여러 프로젝트를 돌며 상태 요약 수집
- 변화가 있는 프로젝트만 선별

### javis가 맡는 일

- 상단장님용 포트폴리오 수준 요약 제공
- 중요한 프로젝트만 먼저 팝업에 올리기

### 장점

- 여러 작업을 동시에 운영할 때 좋습니다.

## 시나리오 9. Browser QA Follow-up

### 추천 타입

- 초반: `thread automation`
- 후반: `thread automation + javis visual supervisor`

### 상황

- 코드는 이미 만들었고, 결과 화면 검수가 필요함

### Codex가 맡는 일

- 최근 작업 문맥과 QA TODO 확인
- 필요 시 후속 작업 제안

### javis가 맡는 일

- 브라우저 화면 읽기
- 시각 증거를 합쳐서 합격 / 보류 / 수정 필요 판단

### 장점

- 이 시나리오가 들어오면 비로소 `javis의 눈`이 진짜 가치가 생깁니다.

## 시나리오 10. Voice Supervisor

### 추천 타입

- 후반부 `thread automation + voice layer`

### 상황

- 상단장님이 말로 “상태 알려줘”, “다음 단계 진행해”, “왜 멈췄어”를 하고 싶음

### Codex가 맡는 일

- underlying work와 thread context 유지

### javis가 맡는 일

- 음성 입력
- 음성 출력
- 상단장님 의도를 Codex follow-up으로 바꿔 전달

### 장점

- 이 시점부터 제품 경험이 진짜 `자비스`에 가까워집니다.

## 지금 당장 우선순위

피디 추천 우선순위는 아래입니다.

1. 마스터플랜 Follow-up Heartbeat
2. Release Smoke 자동 점검
3. PR Babysitting
4. Nightly Project Brief
5. CI Failure Triage

이 5개는 `Codex 네이티브 활용 가치`가 크고, 상단장님 실제 운영 방식과도 가장 잘 맞습니다.

## javis가 늦게 들어와도 되는 영역

- 반복 스케줄링
- 백그라운드 재실행
- 독립 run 분리
- worktree 격리
- 기본 결과 수집

## javis가 빨리 들어와야 하는 영역

- 상단장님용 한국어 운영 UX
- 작은 팝업에서 한 줄 요약
- Codex 결과 + 화면 증거 + 정책을 합친 상위 판단
- 나중의 음성 경험

## 한 줄 결론

초기 `javis`는 모든 실행을 직접 하는 앱보다, `Codex Automations를 가장 영리하게 묶어 주는 감독자`에 더 가깝게 설계하는 것이 맞습니다.
