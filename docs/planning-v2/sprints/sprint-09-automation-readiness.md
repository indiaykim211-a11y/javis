# Sprint 9: Automation Readiness

## 스프린트 목표

automation 사용을 과하게 밀어붙이지 않게 하고, Release 3 핵심 흐름을 반복 검증 가능한 상태로 마감합니다.

## 포함 티켓

- `R3-007` automation safety guard
- `R3-008` Release 3 smoke suite

## 상세 목표

### 1) stop / ask_user 기준 정리

- 언제 automation보다 no automation이 맞는지
- 언제 자동 진행보다 멈춤이 맞는지

### 2) Release 3 빠른 점검 가능성

- mode selector
- thread / project composer
- handoff flow
- runboard
- triage bridge

를 몇 분 안에 점검할 수 있어야 합니다.

## 완료 조건

- automation을 써야 할 때와 말아야 할 때가 제품 안에서 보임
- Release 3 흐름을 5~10분 안에 반복 확인 가능
