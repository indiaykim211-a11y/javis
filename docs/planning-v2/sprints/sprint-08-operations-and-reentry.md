# Sprint 8: Operations and Re-entry

## 스프린트 목표

Codex app에서 automation을 만든 뒤, 결과를 다시 어디서 확인하고 어떻게 다음 행동으로 이어갈지 운영 흐름을 만듭니다.

## 포함 티켓

- `R3-004` launch checklist / handoff flow
- `R3-005` automation runboard
- `R3-006` triage summary bridge

## 상세 목표

### 1) launch flow 정리

- Codex 앱으로 옮겨갈 때 필요한 handoff 절차
- thread / project 별 차이
- 결과를 다시 볼 위치

### 2) 운영 표면 만들기

- 최근 automation 전략
- 최근 결과 확인 메모
- 다음 follow-up 제안

## 완료 조건

- 상단장님이 automation 생성 후 무엇을 해야 하는지 헷갈리지 않음
- Codex 결과를 다시 볼 재진입 흐름이 자연스러움
