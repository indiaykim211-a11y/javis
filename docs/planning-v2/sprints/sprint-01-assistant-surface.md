# Sprint 1: Assistant Surface

## 스프린트 목표

상단장님이 `javis`를 실행했을 때, 더 이상 복잡한 콘솔이 아니라 작은 어시스턴트 팝업을 보게 만듭니다.

## 왜 먼저 하는가

Release 1의 본질은 기능 추가보다 `제품 형태 전환`입니다. 따라서 가장 먼저 바뀌어야 하는 것은 내부 로직이 아니라 메인 사용자 표면입니다.

## 포함 티켓

- `R1-001` Assistant Popup 셸 창
- `R1-002` Surface State/액션 모델
- `R1-003` 팝업 액션 라우팅

## 상세 목표

### 1) 팝업 어시스턴트 존재감 만들기

- 작은 팝업형 창을 별도 계층으로 도입
- 항상 위 표시, 컴팩트 크기, 기본 헤더 구조 확보

### 2) 상태 중심 메인 경험 연결

- 현재 상태
- 최근 판단 이유
- 다음 행동 후보
- 핵심 액션 버튼

이 4개가 메인 팝업에서 살아야 합니다.

### 3) 기존 엔진과 연결

- `진행`, `보류`, `요약`, `설정` 같은 액션이 기존 로직과 연결
- 아직 실제 판단 엔진이 없어도 Surface 동선은 완성

## 완료 조건

- 팝업만 보고도 현재 상태를 대략 이해할 수 있음
- 팝업 버튼이 실제 동작 또는 준비된 동선으로 연결됨
- 기존 콘솔이 메인 첫화면이 아님

## 데모 시나리오

1. javis 실행
2. 작은 팝업 노출 확인
3. 상태/이유/다음 행동 카드 확인
4. `설정`으로 Control Center 진입
5. `진행` 또는 `요약` 동작 확인

## 주요 리스크

- 팝업이 너무 비어 보일 수 있음
- 기존 로직과 분리되며 중복 상태 모델이 생길 수 있음
- 액션이 UI상 존재하지만 실제 연결이 약하면 신뢰가 떨어질 수 있음

## 스프린트 산출물

- Assistant Surface v1
- Surface State v1
- 액션 라우팅 기본형
