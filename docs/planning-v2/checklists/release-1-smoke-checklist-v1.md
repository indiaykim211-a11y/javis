## 수동 체크리스트

### 1. 첫 실행

- 앱이 `작은 팝업`으로 먼저 열린다.
- 첫 화면이 큰 개발자 콘솔처럼 보이지 않는다.
- 상태, 이유, 다음 행동, 버튼 4개 흐름이 바로 읽힌다.

실패 시 먼저 볼 곳:

- 팝업 레이아웃: `app/ui.py`
- 최근 변경: `R1-001`, `R1-007`

### 2. 팝업 compact 동작

- `간단히`를 누르면 팝업이 더 작아진다.
- compact 상태에서 `한눈에 보기` 카드가 보인다.
- `자세히`를 누르면 원래 레이아웃으로 돌아온다.
- 좁은 폭에서도 버튼이 겹치거나 잘리지 않는다.

실패 시 먼저 볼 곳:

- compact 전환: `app/ui.py`
- 버튼 재배치: `app/ui.py`

### 3. Control Center 진입

- 팝업에서 `설정`을 누르면 Control Center가 열린다.
- 열릴 때 `프로젝트` 섹션이 먼저 보인다.
- 팝업으로 돌아와도 앱 흐름이 끊기지 않는다.

실패 시 먼저 볼 곳:

- 창 위치 보정: `app/ui.py`
- 섹션 기본 진입: `app/ui.py`

### 4. 프로젝트 입력과 저장

- `프로젝트 요약`, `목표 수준`, `단계 목록`을 입력할 수 있다.
- `세션 저장` 후 프로젝트 홈 카드가 즉시 갱신된다.
- 저장 시각과 로그 파일 경로가 보인다.

실패 시 먼저 볼 곳:

- 세션 저장: `app/state.py`
- 프로젝트 홈 카드: `app/ui.py`

### 5. 정책 편집기

- `정책` 섹션에서 6개 탭이 보인다.
- `운영 마스터 / 단계 진행 / 화면 판독 / 보정 지시 / 사용자 보고 / 안전 규칙` 구조를 한눈에 이해할 수 있다.
- 텍스트를 고치면 `저장 전 변경 있음` 상태가 보인다.
- `현재 섹션 기본값 복원`과 `전체 기본값 복원`이 동작한다.

실패 시 먼저 볼 곳:

- 정책 모델: `app/models.py`
- 정책 탭 UI: `app/ui.py`

### 6. 최근 프로젝트 복원

- 서로 다른 프로젝트를 2개 저장하면 최근 프로젝트 목록이 쌓인다.
- 이전 프로젝트를 선택해 `선택한 프로젝트 이어가기`를 누르면 편집 값과 단계 진행 정보가 돌아온다.

실패 시 먼저 볼 곳:

- recent project 저장/복원: `app/state.py`
- recent project UI: `app/ui.py`

### 7. 핵심 버튼 동선

- 팝업의 기본 버튼 4개가 비어 있지 않다.
- `보류`를 누르면 보류 상태로 바뀐다.
- `재개`를 누르면 다시 진행 가능 상태로 돌아온다.
- `요약`은 상태 이해를 돕는 방향으로 동작한다.

실패 시 먼저 볼 곳:

- 상태 모델: `app/services/workflow.py`
- 팝업 액션 라우팅: `app/ui.py`

## 통과 기준

- 팝업이 메인 제품처럼 느껴진다.
- Control Center는 설정/운영 창 역할로 분리되어 보인다.
- 프로젝트 저장, 최근 프로젝트 복원, 정책 편집 흐름이 모두 막히지 않는다.
- compact 동작이 어색하지 않다.

## 산출물 기록 추천

- 사용한 워크스페이스
- 통과 / 보류 / 실패
- 실패 시 재현 단계 1~3줄
