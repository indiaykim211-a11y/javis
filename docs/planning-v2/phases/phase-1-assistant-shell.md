# Phase 1: Assistant Shell

## 목표

`javis`의 사용자 표면을 `작은 데스크톱 어시스턴트`로 재구성합니다.

이 단계의 핵심은 자동화 능력 자체보다도, 상단장님이 자연스럽게 믿고 부를 수 있는 형태를 먼저 만드는 것입니다.

## Phase 1의 정의

이 단계가 끝나면 `javis`는 아래처럼 보여야 합니다.

- 평소에는 작은 팝업으로 존재
- 지금 무슨 상태인지 짧게 설명
- 복잡한 설정은 뒤로 숨김
- 다음 릴리즈에서 판단과 화면 인식이 들어올 자리가 준비됨

## 세부 목표

### 1) Surface와 Control Center 분리

- 메인 화면은 `Assistant Surface`
- 고급 제어는 `Control Center`
- 둘의 책임을 섞지 않음

### 2) 상태 중심 메인 경험

- 메인 화면에서 보이는 것은 `상태`, `이유`, `다음 행동`이어야 함
- 내부 구현 세부사항이 전면에 나오면 안 됨

### 3) 복원 가능한 일상 사용 흐름

- 앱을 다시 켜도 이어서 쓰는 느낌이 나야 함
- “최근 프로젝트 다시 열기”가 자연스러워야 함

### 4) 이후 릴리즈를 위한 자리 확보

- OpenAI 판단 결과 슬롯
- 비전 판독 결과 슬롯
- 음성 버튼/상태 슬롯

## 사용자 관점 핵심 질문

Phase 1은 아래 질문에 답해야 합니다.

- 지금 `javis`가 뭘 하고 있나요?
- 왜 멈춰 있나요?
- 내가 지금 뭘 누르면 되나요?
- 복잡한 설정은 어디서 하나요?

## 핵심 화면 구조

### Assistant Surface

작은 팝업 중심 화면입니다.

- 헤더
  - 상태 배지
  - 프로젝트명 또는 요약
- 본문
  - 현재 상태 한 줄
  - 최근 판단 이유
  - 다음 행동 후보
- 액션 바
  - `진행`
  - `보류`
  - `요약`
  - `설정`
- 보조 슬롯
  - 나중에 `말하기`, `듣는 중`, `화면 읽는 중` 표시 가능

### Control Center

상세 설정과 운영용 화면입니다.

- 프롬프트 정책
- Codex 타겟 설정
- 캘리브레이션
- 세션/최근 프로젝트
- 로그/스크린샷
- 고급 자동화 옵션

Control Center의 정보구조 상세는 `../ux/control-center-ia-v1.md`를 기준으로 설계합니다.

## 이 단계에서 필요한 상태 모델

### 1) Surface State

메인 팝업 표시 전용 상태입니다.

- 현재 상태 배지
- 요약 문장
- 최근 판단 이유
- 추천 행동
- 위험 레벨

### 2) Session State

기존 저장 상태를 메인 경험에 맞게 연결합니다.

- 현재 프로젝트
- 현재 단계 인덱스
- 마지막 저장 시각
- 최근 프로젝트 목록

### 3) Capability State

현재 엔진 연결 상태를 보여주는 상태입니다.

- Codex 연결 상태
- 판단 엔진 연결 상태
- 화면 인식 준비 상태
- 음성 준비 상태

## 디자인 원칙

### 1) 작지만 고급스럽게

- 지나치게 도구적이거나 촌스러우면 안 됨
- 작은 크기에서도 제품다운 인상이 나야 함

### 2) 결론 우선

- 상태를 설명할 때는 먼저 결론
- 세부 이유는 한 단계 아래

### 3) 부담 없는 존재감

- 항상 켜둘 수 있어야 함
- 화면을 크게 점유하지 않아야 함

## 구현 원칙

### 1) 기존 엔진 재사용

- 현재까지 만든 세션 저장/복원
- 단계 큐
- 프롬프트 프리뷰
- 기존 Codex 제어 엔진

이런 요소는 버리지 않고 `Control Center`로 이동하거나 숨깁니다.

### 2) 새 메인 표면 추가

- 기존 콘솔을 억지로 축소하지 않음
- 메인 UX를 위한 전용 화면 계층을 새로 둠

### 3) Release 2 대비 슬롯 설계

- 판단 결과 카드
- 추천 액션 카드
- OpenAI 응답 상태

## 완료 기준

1. 메인 팝업이 제품의 주 화면 역할을 함
2. Control Center는 보조 화면으로 물러남
3. 세션 복원과 최근 프로젝트 접근이 메인 흐름을 방해하지 않음
4. 다음 단계인 `Codex-First Foundation`을 얹기 좋은 구조가 됨
