# 아키텍처 메모

## 목표

상단장님의 현재 역할인 아래 4가지를 javis가 대신하도록 만드는 것이 목표입니다.

- Codex 작업이 끝났는지 확인
- 결과 화면을 살펴보고 이상 여부 판단
- 이상이 없으면 다음 단계 지시
- 문제가 있으면 멈추고 재지시

## 왜 바로 완전 자동화가 아니라 MVP부터 가는가

Codex 데스크톱 앱은 내부 렌더링이 커스텀일 수 있어서 일반적인 Windows 접근성 트리에서 본문 텍스트가 잘 안 보일 수 있습니다. 따라서 초반에는 다음 전략이 안전합니다.

- UIA로 가능한 것은 최대한 사용
- 화면 캡처를 기본 관측 수단으로 사용
- 사람 대신 다음 단계 버튼 역할부터 자동화
- 판단 로직은 보수적으로 설계

## MVP 구조

- `app/ui.py`
  - Tkinter 기반 데스크톱 UI
- `app/automation/windows_ui.py`
  - Windows 창 조회, 포커스, 캡처, 클릭, 붙여넣기 전송
- `app/automation/bmp.py`
  - BMP 스크린샷 시그니처 계산
- `app/services/workflow.py`
  - 자동 루프와 단계 전송 상태 머신
- `app/state.py`
  - 세션 설정 저장/복원

## 자동 진행 기준

MVP에서는 아래 조건을 모두 만족할 때만 다음 단계를 보냅니다.

- Codex 창을 찾을 수 있음
- 최근 스크린샷과 현재 스크린샷이 충분히 비슷함
- 설정한 횟수만큼 연속으로 비슷함
- 마지막 전송 이후 쿨다운이 지남
- 남은 단계가 있음

## 2단계 확장

- OCR 또는 비전 API로 Codex 대화/브라우저 결과 읽기
- 코드 변경 이후 웹 페이지가 기준에 맞는지 자동 점검
- 문제 발견 시 정해진 수정 프롬프트 자동 삽입
- 로그/스크린샷/판정 사유 저장
